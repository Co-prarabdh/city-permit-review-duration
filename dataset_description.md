# City Permit Review Duration Dataset

## Overview

This is a synthetic municipal permit dataset designed for a tabular regression benchmark. Each row represents a building permit application at the time it is submitted to a city permitting department. The target variable is `review_duration_days`, the number of calendar days between submission and the first completed review decision.

The dataset is generated from a reproducible script rather than copied from a public competition. It includes numerical, categorical, boolean, date, free-text, and pipe-separated multi-label fields. It also intentionally includes post-submission workflow columns that are not available at prediction time, so strong solutions must identify and remove target leakage.

## File Structure

- `train.csv` - training set with labels
- `test.csv` - test set without `review_duration_days`
- `sample_submission.csv` - example submission with the required columns

The raw creator upload contains:

- `raw/data.csv` - full synthetic source data before splitting

## Features

| Column | Type | Description |
|---|---|---|
| `permit_id` | string | Unique permit application identifier |
| `submitted_date` | date | Application submission date in `dd/mm/yyyy` format |
| `permit_type` | categorical | Permit type, such as alteration, new building, demolition, solar, or trade-specific work |
| `project_category` | categorical | Residential, commercial, mixed-use, civic, or industrial project category |
| `borough` | categorical | City district where the project is located |
| `zoning_district` | categorical | Planning/zoning district code |
| `applicant_type` | categorical | Owner, contractor, architect, engineer, or permit expediter |
| `contractor_tier` | categorical | Synthetic historical contractor tier |
| `project_description` | string | Short free-text description of the project scope |
| `estimated_cost_usd` | float | Declared project cost |
| `planned_floor_area_sqft` | float | Planned project floor area |
| `residential_units` | integer | Number of residential units affected or created |
| `commercial_sqft` | integer | Commercial square footage affected |
| `building_age_years` | integer | Age of the existing building |
| `prior_permits_5yr` | integer | Prior permit count at the same property in the last five years |
| `open_violations_count` | integer | Open property violations at submission time |
| `flood_zone` | boolean | Whether the project is in a flood zone |
| `landmark_district` | boolean | Whether the project is in a landmark or historic district |
| `expedited_requested` | boolean | Whether expedited review was requested |
| `docs_submitted_count` | integer | Count of submitted supporting documents |
| `missing_document_count` | integer | Count of required documents missing at intake |
| `requested_trades` | string | Pipe-separated list of trades included in the project scope |
| `required_reviews` | string | Pipe-separated list of review disciplines expected at intake |
| `reviewer_team` | categorical | Internal reviewer team assigned after submission; leakage |
| `reviewer_assigned_date` | date | Date a reviewer was assigned; leakage |
| `first_response_date` | date | Date of first response to the applicant; leakage |
| `revision_cycles` | integer | Number of revision cycles after review begins; leakage |
| `inspection_count` | integer | Downstream inspection count; leakage |
| `final_status` | categorical | Final permit outcome; leakage |
| `review_duration_days` | integer | Target review duration in calendar days |

## Data Characteristics

- Target distribution: right-skewed, with most permits reviewed quickly and a long tail for complex projects.
- Date features: `submitted_date`, `reviewer_assigned_date`, and `first_response_date` use `dd/mm/yyyy`.
- Text feature: `project_description` contains scope words that correlate with complexity.
- Concatenated categoricals: `requested_trades` and `required_reviews` contain pipe-separated values.
- Leakage columns: `reviewer_team`, `reviewer_assigned_date`, `first_response_date`, `revision_cycles`, `inspection_count`, and `final_status` are generated after submission and should not be used for an at-submission prediction model.
- Missing values: a small amount of missingness is present in `contractor_tier`, `zoning_district`, `project_description`, and `estimated_cost_usd`.
